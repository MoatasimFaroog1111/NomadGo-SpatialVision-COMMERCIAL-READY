# دليل إعداد الأسرار في GitHub وإدارة متغيرات البيئة بشكل آمن

تعتبر إدارة الأسرار ومتغيرات البيئة جزءًا أساسيًا من تطوير البرمجيات. في هذا الدليل، سنستعرض كيفية إعداد الأسرار في GitHub وكيفية إدارتها بأمان.

## 1. ما هي الأسرار (Secrets)؟
الأسرار هي معلومات حساسة مثل رموز الوصول وكلمات المرور، والتي لا يجب أن تكون مرئية في الكود الخاص بك. في GitHub، يمكنك إعداد الأسرار لحماية هذه المعلومات.

## 2. إعداد الأسرار في GitHub
لتكوين الأسرار في GitHub، اتبع الخطوات التالية:

### الخطوة 1: افتح إعدادات المستودع
- انتقل إلى مستودع GitHub الخاص بك.
- انقر على علامة تبويب "Settings".

### الخطوة 2: الوصول إلى قسم الأسرار
- في قائمة الجانب، ابحث عن "Secrets and variables".
- انقر على "Actions".

### الخطوة 3: إنشاء سر جديد
- انقر على زر "New repository secret".
- أدخل اسم السر (مثل `MY_SECRET_TOKEN`) وقيمته.
- انقر على "Add secret" لحفظ السر.

## 3. استخدام الأسرار في GitHub Actions
لاستخدام الأسرار في GitHub Actions، يمكنك الإحالة إليها باستخدام الصيغة التالية:

```yaml
      env:
        MY_SECRET_TOKEN: ${{ secrets.MY_SECRET_TOKEN }}
```

## 4. إدارة متغيرات البيئة
متغيرات البيئة هي معلومات ضرورية لتشغيل البرامج. يمكن إعدادها محليًا أو في بيئات السحابة:

### على مستوى النظام:
- يمكنك ضبط متغيرات البيئة في النظام الخاص بك باستخدام سطر الأوامر أو إعدادات النظام.

### في GitHub Actions:
يمكن أيضًا إعداد متغيرات البيئة في ملف YAML الخاص بإجراءات GitHub:

```yaml
      env:
        NODE_ENV: production
```

## 5. نصائح لتأمين الأسرار ومتغيرات البيئة
- تجنب تسجيل الأسرار في السجل.
- استخدم أسرار GitHub بدلاً من hardcoding القيم السرية في الكود.
- قم بمراجعة الأسرار بانتظام وإزالة غير المستخدمة.

باتباع هذه الإرشادات، يمكنك إدارة الأسرار ومتغيرات البيئة بفعالية وأمان.
