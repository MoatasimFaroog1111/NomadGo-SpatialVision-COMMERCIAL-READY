## 2. SDK & NDK Installation

### Via Android Studio SDK Manager

1. Open Android Studio → **Settings** → **Languages & Frameworks** → **Android SDK**
2. **SDK Platforms** tab:
   - Check **Android 14.0 (API 34)**
   - Expand it and ensure **Android SDK Platform 34** is checked
3. **SDK Tools** tab:
   - Check **Android SDK Build-Tools 34.0.0**
   - Check **NDK (Side by side)** — then click **Show Package Details** and select **25.2.9519653**
   - Check **Android SDK Command-line Tools (latest)**
   - Check **Android SDK Platform-Tools**
   - Check **CMake 3.22.1** (required by tflite_flutter native compilation)
4. Click **Apply** and wait for downloads to complete

### Via Command Line (sdkmanager)

```bash
# Set ANDROID_HOME if not already set
export ANDROID_HOME=$HOME/Android/Sdk    # Linux/macOS
# Windows: set ANDROID_HOME=%LOCALAPPDATA%\Android\Sdk

# Install required packages
sdkmanager "platforms;android-34"
sdkmanager "build-tools;34.0.0"
sdkmanager "ndk;25.2.9519653"
sdkmanager "platform-tools"
sdkmanager "cmdline-tools;latest"
sdkmanager "cmake;3.22.1"

# Accept all licenses
sdkmanager --licenses
```

### Verify Installation

```bash
# Check SDK location
echo $ANDROID_HOME
# Should show: /Users/<you>/Android/Sdk (macOS) or /home/<you>/Android/Sdk (Linux)

# Verify NDK exists
ls $ANDROID_HOME/ndk/25.2.9519653/
# Should list: build, meta, ndk-build, platforms, prebuilt, etc.

# Verify build tools
ls $ANDROID_HOME/build-tools/34.0.0/
```

## 4. YOLOv8 Model Preparation

The app requires a YOLOv8n model in TensorFlow Lite format.

### Export from Ultralytics

```bash
# Create a Python virtual environment (recommended)
python3 -m venv yolo_env
source yolo_env/bin/activate    # Linux/macOS
# yolo_env\Scripts\activate     # Windows

# Install ultralytics
pip install ultralytics

# Export YOLOv8n to TFLite (float32, 320×320)
yolo export model=yolov8n.pt format=tflite imgsz=320

# The output file will be at:
# runs/detect/export/yolov8n_float32.tflite
# or: yolov8n_saved_model/yolov8n_float32.tflite (varies by version)
```

### Place the Model

```bash
# Copy to project assets
cp yolov8n_float32.tflite /path/to/scan3d_vision/assets/models/yolov8n.tflite
```

### Verify Model Specifications

The app expects:

| Property | Value |
|----------|-------|
| Input shape | `[1, 320, 320, 3]` |
| Input type | float32, normalized [0, 1] |
| Output shape | `[1, 84, 2100]` |
| Output format | 4 bbox coords + 80 class scores × 2100 anchors |

To verify with Python:

```python
import tensorflow as tf

interpreter = tf.lite.Interpreter(model_path="yolov8n.tflite")
interpreter.allocate_tensors()

input_details = interpreter.get_input_details()
output_details = interpreter.get_output_details()

print(f"Input:  {input_details[0]['shape']} {input_details[0]['dtype']}")
print(f"Output: {output_details[0]['shape']} {output_details[0]['dtype']}")
# Expected:
# Input:  [1, 320, 320, 3] <class 'numpy.float32'>
# Output: [1, 84, 2100] <class 'numpy.float32'>
```

## 6. ARCore Setup

### AndroidManifest.xml (already configured)

The manifest at `android/app/src/main/AndroidManifest.xml` includes:

```xml
<!-- ARCore required -->
<uses-feature android:name="android.hardware.camera.ar" android:required="true"/>

<application>
    <meta-data android:name="com.google.ar.core" android:value="required"/>
</application>
```

- `android:value="required"` means the app will only appear in Play Store for ARCore-supported devices
- Change to `"optional"` if you want to gracefully degrade on non-AR devices

### ARCore Dependency

The `ar_flutter_plugin_2` package automatically adds the ARCore dependency via its own `build.gradle`. No manual ARCore dependency is needed in your app-level `build.gradle`.

### Sceneform (Optional)

If you need 3D object rendering via ARCore Sceneform:

```groovy
// In android/app/build.gradle, add to dependencies block:
dependencies {
    implementation 'com.google.ar.sceneform.ux:sceneform-ux:1.17.1'
    implementation 'com.google.ar.sceneform:core:1.17.1'
}
```

> **Note**: Our project uses `flutter_3d_controller` for 3D preview, so Sceneform is not required.

### Verify ARCore on Device

Before first run, ensure Google Play Services for AR is installed:

```bash
adb shell pm list packages | grep arcore
# Should show: package:com.google.ar.core
```

If not installed, it will auto-install from Play Store when the app launches, or install manually:
https://play.google.com/store/apps/details?id=com.google.ar.core

## 7. TFLite Native Library Setup

### Automatic Setup (Default)

The `tflite_flutter: ^0.12.1` package handles native library inclusion automatically via its Gradle plugin. When you run `flutter pub get`, it downloads the prebuilt `libtensorflowlite_jni.so` for all supported ABIs.

### Manual Setup (If Automatic Fails)

If `tflite_flutter` fails to resolve native libraries:

```bash
# Install the tflite_flutter helper tool
dart pub global activate tflite_flutter_helper

# Download native libraries
cd /path/to/scan3d_vision
dart run tflite_flutter:install
```

This places `.so` files in `android/app/src/main/jniLibs/<abi>/`:

```
android/app/src/main/jniLibs/
├── arm64-v8a/
│   └── libtensorflowlite_jni.so
├── armeabi-v7a/
│   └── libtensorflowlite_jni.so
└── x86_64/
    └── libtensorflowlite_jni.so
```

### ABI Filtering (Reduce APK Size)

To build only for ARM64 devices (most modern phones):

```groovy
// In android/app/build.gradle, inside defaultConfig:
defaultConfig {
    // ... existing config
    ndk {
        abiFilters 'arm64-v8a'  // 64-bit ARM only
        // Add 'armeabi-v7a' for 32-bit ARM support
        // Add 'x86_64' for Intel emulator support
    }
}
```

### NNAPI Delegate (Hardware Acceleration)

For best inference performance on Android, the app uses NNAPI by default (configured in `yolo_detector.dart`). NNAPI requires:
- Android 8.1+ (API 27+) — our minSdk 24 will fall back to CPU on API 24–26
- Device with supported neural processing hardware

## 10. Signing Configuration

### Debug Signing (Development)

Debug builds automatically use the debug keystore at `~/.android/debug.keystore`. No configuration needed.

### Release Signing (Production)

#### Step 1: Generate a Keystore

```bash
keytool -genkey -v \
  -keystore scan3d-release-key.jks \
  -storetype JKS \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000 \
  -alias scan3d

# You will be prompted for:
# - Keystore password
# - Key password
# - Name, organization, location info
```

#### Step 2: Create `android/key.properties`

```properties
storePassword=<your-keystore-password>
keyPassword=<your-key-password>
keyAlias=scan3d
storeFile=../scan3d-release-key.jks
```

> **IMPORTANT**: Add `key.properties` and `*.jks` to `.gitignore` — never commit signing keys.

#### Step 3: Reference in `build.gradle`

Update `android/app/build.gradle`:

```groovy
// Add above the android { } block:
def keystoreProperties = new Properties()
def keystorePropertiesFile = rootProject.file('key.properties')
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
}

android {
    // ... existing config

    signingConfigs {
        release {
            keyAlias keystoreProperties['keyAlias']
            keyPassword keystoreProperties['keyPassword']
            storeFile keystoreProperties['storeFile'] ? file(keystoreProperties['storeFile']) : null
            storePassword keystoreProperties['storePassword']
        }
    }

    buildTypes {
        release {
            signingConfig signingConfigs.release  // Changed from signingConfigs.debug
            minifyEnabled true
            shrinkResources true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
}
```

## 13. Troubleshooting

### Build Errors

#### `NDK not configured` or `NDK version mismatch`

```
Error: NDK not configured. Download it with SDK manager.
Preferred NDK version is "25.2.9519653"
```

**Fix**: Install the exact NDK version:

```bash
sdkmanager "ndk;25.2.9519653"
```

Or set in `android/app/build.gradle`:

```groovy
ndkVersion "25.2.9519653"
```

#### `Kotlin version too low for AGP`

```
The Android Gradle plugin supports only Kotlin Gradle plugin version 1.5.20 and higher
```

**Fix**: Update `ext.kotlin_version` in `android/build.gradle`:

```groovy
ext.kotlin_version = '1.9.22'
```

#### `Could not determine the dependencies of task ':app:compileDebugJavaWithJavac'`

**Fix**: This usually means AGP/Gradle version mismatch. Ensure:

```
# gradle-wrapper.properties
distributionUrl=https\://services.gradle.org/distributions/gradle-8.3-all.zip

# build.gradle
classpath 'com.android.tools.build:gradle:8.1.4'
```

#### `Execution failed for task ':app:mergeDebugNativeLibs'` (TFLite)

**Fix**: The tflite native libraries may conflict. Add to `android/app/build.gradle`:

```groovy
android {
    packagingOptions {
        pickFirst 'lib/*/libtensorflowlite_jni.so'
        pickFirst 'lib/*/libtensorflowlite_gpu_jni.so'
    }
}
```

#### `Minimum SDK version 24 required` errors from dependencies

**Fix**: Ensure all dependency minSdk requirements are met. Our `minSdkVersion 24` satisfies:

- `camera`: requires 21+
- `ar_flutter_plugin_2`: requires 24+
- `tflite_flutter`: requires 21+
- `permission_handler`: requires 21+

#### `Manifest merger failed` — duplicate permissions

**Fix**: Add `tools:replace` to the manifest:

```xml
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools">

    <uses-permission android:name="android.permission.CAMERA"
        tools:replace="android:required"/>
</manifest>
```

### Runtime Errors

#### ARCore not working — "ARCore is not supported on this device"

- Verify device is on the [ARCore supported devices list](https://developers.google.com/ar/devices)
- Install/update Google Play Services for AR from Play Store
- Ensure camera permissions are granted

#### TFLite model fails to load

```
Exception: Failed to load model
```

**Checklist**:

1. File exists at `assets/models/yolov8n.tflite`
2. Declared in `pubspec.yaml` under `flutter: assets:`
3. Run `flutter clean && flutter pub get` to refresh asset bundling
4. Verify model shape matches expectations (see Section 4)

#### App crashes on launch — `PlatformException`

- Run with verbose logging: `flutter run --debug --verbose`
- Check logcat: `adb logcat -s flutter,ARCore,TfLite`
- Common cause: missing `CAMERA` permission — ensure it's in AndroidManifest.xml

#### Camera black screen

- Grant camera permission manually: **Settings** → **Apps** → **Scan3D Vision** → **Permissions** → **Camera**
- Ensure no other app is using the camera
- Test with: `adb shell am start -a android.media.action.STILL_IMAGE_CAMERA`

#### ProGuard stripping needed classes (release build works, release crashes)

- Test with ProGuard disabled first:

```groovy
buildTypes {
    release {
        minifyEnabled false  // Temporarily disable
    }
}
```

- If it works with ProGuard disabled, check logcat for `ClassNotFoundException` and add keep rules
- Run: `adb logcat | grep -i "ClassNotFound\|NoSuchMethod\|proguard"`

### Performance Issues

#### Low FPS during scanning

1. Reduce model input size in `lib/app/constants.dart`:
   ```dart
   static const int modelInputSize = 256; // Down from 320
   ```
2. Enable NNAPI delegate in `lib/services/yolo_detector.dart`
3. Reduce camera resolution in `lib/services/camera_service.dart`:
   ```dart
   ResolutionPreset.medium  // Instead of ResolutionPreset.high
   ```

#### Large APK size

- Use `--split-per-abi` to generate per-architecture APKs
- Add ABI filter to limit architectures (see Section 7)
- Use App Bundle (AAB) for Play Store — Google handles ABI splitting
